## Overview of Codebase

### Theme Configuration

**Location**: `frontend/src/styles/globals.css`

The project uses a sophisticated theming system:
- **Color Space**: OKLCH (modern, perceptually uniform color space)
- **Dark Mode**: Fully configured with `.dark` class variant
- **CSS Variables**: Complete design token system with semantic naming

Key color tokens relevant to the layout:
- `--background`: oklch(1 0 0) in light, oklch(0.145 0 0) in dark
- `--card`: Slightly different from background, suitable for header/footer
- `--foreground`: Text colors
- `--border`: oklch(0.922 0 0) in light, oklch(1 0 0 / 10%) in dark
- `--sidebar`: Dedicated sidebar color scheme (for future use)

Base styling (lines 113-120):
```css
@layer base {

```

## Gotchas and Pitfalls

### 1. Missing CSS Import (CRITICAL)
**Issue**: `src/main.tsx` line 8 imports `./index.css`, but this file doesn't exist!

```tsx
import './index.css'  // ❌ File doesn't exist
```

The actual global styles are in `./styles/globals.css`.

**Impact**: Global styles and theme may not be loading properly.

**Fix Required**: Update the import to:
```tsx
import './styles/globals.css'
```

### 2. Dual Router Setup (Architectural Decision Needed)
**Issue**: The project has both:
- `routes/__root.tsx` (file-based routing, only 3 lines, minimal)
- `lib/router.tsx` (manual route definition, currently active)

The manual router in `lib/router.tsx` is currently being used, not the file-based `__root.tsx`.

**Decision**: Since manual routing is active, integrate GlobalLayout into `lib/router.tsx` by updating the rootRoute component.

### 3. Unused Legacy Files
- `App.tsx` and `App.css` are remnants from Vite scaffolding and aren't used
- Can be safely ignored or deleted

### 4. Tailwind CSS v4 Syntax
**Important**: The project uses Tailwind CSS v4 with new syntax:
- Uses `@import "tailwindcss"` instead of old directives
- Uses new `@custom-variant` for dark mode
- Some older Tailwind patterns may not work

**Impact**: Ensure any examples or patterns used are v4-compatible.

### 5. Color Scheme Recommendations
For header/footer differentiation from body:

**Recommended approach**:
- **Header/Footer**: Use `bg-card` (same for both)
- **Body**: Uses `bg-background` (via body element)
- **Borders**: Use `border-border` for separation

This follows the existing design token system and provides subtle differentiation while maintaining visual harmony.

**DO NOT** hardcode colors - use semantic tokens for dark mode support.

### 6. TanStack Router Integration Point
**Where to add GlobalLayout**:

Update the `rootRoute` component in `src/lib/router.tsx` (lines 8-15):

```tsx
// Before (current):
const rootRoute = createRootRoute({
  component: () => (
    <div className="min-h-screen bg-background text-foreground">
      <Outlet />
      {import.meta.env.DEV && <TanStackRouterDevtools position="bottom-right" />}
    </div>
  ),
})

// After (with GlobalLayout):
const rootRoute = createRootRoute({
  component: () => (
    <>
      <GlobalLayout>
        <Outlet />
      </GlobalLayout>
      {import.meta.env.DEV && <TanStackRouterDevtools position="bottom-right" />}
    </>
  ),
})
```

### 7. Component Naming Convention
Follow the project's naming patterns from tech-brief.md:
- **File name**: `GlobalLayout.tsx` (PascalCase)
- **Component name**: `GlobalLayout` (matches file name)
- **Location**: `src/components/GlobalLayout.tsx`

### 8. Testing Expectations
From instructions.md, the layout must ensure:
- Header always at the top
- Footer always at the bottom
- Body fills space between header and footer
- Header and footer share the same background color
- Header/footer background differs from body and global background

**Implementation tip**: Use CSS Grid or Flexbox with `min-h-screen` to ensure footer stays at bottom even with minimal content.
