# Task: Note Editor

## Description
This is an instruction for note editor attached to each section in planner editor:
This document describes how to build a Reddit-like WYSIWYG post editor using Tiptap. The editor supports text formatting, image uploads to Cloudflare, and a focus-based preview mode where unfocused editors display as read-only previews. OVERVIEW The editor is a controlled React component that can be embedded multiple times on a single page. Each editor instance manages its own title, rich text content, and images. When focused, users see the full editing toolbar. When unfocused, the editor displays as a clean preview matching the final post appearance. Content is stored as JSON and submitted to a Spring Boot backend along with other form data. CORE TECHNOLOGY Tiptap is a headless rich text editor built on ProseMirror. It provides a React integration through the useEditor hook and EditorContent component. You should research the Tiptap documentation at tiptap.dev to understand how extensions work, since all formatting features like bold, italic, and images are added through extensions. The StarterKit extension bundles common features together. Understanding the difference between nodes and marks is important because nodes are block-level elements like paragraphs and images while marks are inline formatting like bold and links. COMPONENT STRUCTURE The main PostEditor component accepts a value prop containing the current content and an onChange callback to notify the parent of updates. This follows React's controlled component pattern. Inside, you have a title input field and the Tiptap editor wrapped in EditorContent. The toolbar appears when the editor is focused and hides when unfocused. Focus state is managed with standard React state and onFocus and onBlur event handlers on the editor container. TEXT FORMATTING FEATURES The editor supports bold, italic, strikethrough, three heading levels, bullet lists, numbered lists, block quotes, inline code, fenced code blocks, hyperlinks, and spoiler tags. Most of these come from StarterKit. Links require the separate Link extension. Spoilers need a custom extension you will create by studying how existing mark extensions work in Tiptap. IMAGE UPLOAD APPROACH Images are uploaded to Cloudflare and the returned URL is inserted into the editor. Since Cloudflare is not set up yet, you will create an adapter interface that defines upload and delete methods. A mock adapter using browser blob URLs will work during development. When Cloudflare is ready, you create a real adapter that implements the same interface. Research Cloudflare Images for a simpler managed service or Cloudflare R2 for more control with S3-compatible APIs. The Direct Creator Upload flow lets browsers upload directly to Cloudflare without proxying through your backend. FOCUS AND PREVIEW BEHAVIOR When the editor loses focus, you set the Tiptap editable option to false and hide the toolbar. This makes the editor read-only and appear as the final post would look. When the user clicks back into the editor, you restore editable to true and show the toolbar again. Research how to dynamically toggle the editable property on an existing Tiptap editor instance. PERFORMANCE WITH MULTIPLE EDITORS Having 20 editors on one page means 20 Tiptap instances in memory even when unfocused since setting editable to false does not destroy the editor. Start by implementing all 20 editors normally and test performance. If initial load is slow, research lazy initialization where you only create the Tiptap instance when a section is first focused and render static HTML for never-touched sections. The generateHTML function from the tiptap html package converts JSON content to HTML without needing a live editor. If memory becomes a problem, research using a single shared editor instance that swaps content when users switch between sections. DATA FORMAT Tiptap stores content as a JSON structure called JSONContent. This is better than HTML for storage because it is smaller and easier to manipulate. When submitting the form, each editor provides its JSON content which you include in the payload to your Spring Boot backend. Research how getJSON and setContent methods work on the Tiptap editor. LINK INSERTION When users click the link button, a dialog appears asking for the URL and optional display text. If text was selected before clicking, that becomes the default display text. Research the shadcn Dialog component for the modal and Tiptap's setLink command for applying links to content. THINGS TO RESEARCH Read the Tiptap getting started guide and understand how useEditor hook works. Study the extensions architecture and how StarterKit bundles multiple extensions. Look at examples of custom mark extensions to prepare for building the spoiler feature. Understand JSONContent structure since this is your storage format. Compare Cloudflare Images versus R2 to decide which suits your needs. Review how to make Tiptap work as a controlled component with external value updates. Test mounting multiple Tiptap editors to understand real-world performance. Look into generateHTML for rendering content without a live editor instance.
