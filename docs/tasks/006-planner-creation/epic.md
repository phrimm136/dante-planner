# Epic: Planner Creation

## Description
I want to create a page that create/edit a plan for Limbus Company Mirror Dungeon run. It includes category, name, identity/ego building section, start buff section, three ego gift sections, and theme pack - floor specific ego gift section.

The category and name are in the name div. Left is category, and right for name. The category includes 5F, 10F, and 15F, which should be i18n-compatible. The name is a plain text box.

Identity/ego building section, or deck building section, includes sinner-wise identity and ego viewer. Each sinner's card is composed with a exisitng id card, id's skill 1/2/3 afffinity-type viewer, and five ego viewer, in vertical order. Each skill is composed with skill type icon with a background color corresponding to the skill affinity. Each ego is composed with the circular ego image with a background color correpsonding to the ego affinity. Below the viewer, there exists status viewer. The status viewer consists of affinity EA viewer and keyword EA viewer, in vertical order The affinity EA viewer calculates the selected identities skill affinity and egos affinity cost. The keyword viewer calculates the identities with certain keywords. Below the status viewer, there exists an identity/ego list. The list is already implemented, so use that. Note that there must be a toggle between identity and ego so that identity and ego list are swapped each other. And there must be uptie/threadspin tier selector (1-4) to give the users freedum of selection. Identity list is set to be default. In list, clikcing one of identity or ego changes the corresonding sinner's identity or ego. Note that if the current ego with a certain type is empty, fill with the selection. If not, modify the equipped ego. For example, if you equipped a HE ego and click another HE ego, change the HE ego equipment. If a sinner's HE ego equipment is empty, clicking a HE ego fills the equipment slot. The default identity and ego ends with the id {3-digit}01. Also, the equipped id and ego must have indicator over the cards so that the user can notice they are selected ones, and must be on the top of the list when reloaded. Now look at the viewer. Clicking one sinner card allocates/deallocates the deploy order. That is, if you click the sinner 1 and 3, the deploy order of them is #1 and #2. Then, clicking the sinner 1 again deallocates the order and then reallocate the sinner 3's order to #1. There are 7 (editable) deployment and 5 (rest of 12 sinners) backups. For example, the #1-#7 order is deployment, and #8-#12 is backup. At the bottom there are buttons for importing from deck code, copying deck code to clipboard, and resetting the deployment order.

The start buff section includes ten cards with icon image on the upper-left, buff name on the upper-right, buff description in the middle, and two enhancement modification buttons on the bottom, on for one enhencement (+) and another for two enhencement (++). Note that the buttons are mutual exclusive. With no buttons selected, a buff description shows no enhencement description. With one enhencement (left button selected) a plus sign is appended to the name and the description show one enhencement description. With two enhencement (left button selected) two plus signs are appended to the name and the description show two enhencement description. User can select the buffs freely. On the upper-right of the buff list, there is a cost indicator. The indicator calculates the total cost of the selected buff and those enhencement, showing the cost to the user.

There are three ego gift sections, one for start ego gift, another for observed ego gift, and the other for comprehensive ego gift list.

The start ego gift list has the dedicated viewer. There are 11 keyword list in grid with three gift in vertical order. Only one of them can be selected. With the selected keyword, 1-3 gifts within the keyword can be selected. The number of selection is determined by the start buff.

The observed ego gift list has the implemented ego gift list at the left and observed gift viewer on the right. The list is the same as that in the `/egogift` page, but it takes an id list as argument, and clicking one of the gifts puts its id into 3-length list. Clicking again removes its id from the 3-length list. The gifts in the 3-length list is highlighted. The viewer takes 3-length list as argument and shows gifts in the list. Clicking a gift in the viewer remove it from the 3-length list.

The skill EA list shows sinner-wise skill EAs. For each sinner, a card contains id profile and skill EA viewer in vertical order. The skill EA viewer consists of name S1, S2, and S3, each of which has the number of the skill (3/2/1). Clicking a card shows a pane for skill swap. There are for sections, S1 -> S2, S2->S3, S2->S3, and reset. Clicking the first three section changes the number of correpsonding skills (decrementing the former one and incrementing the latter one)

The comprehensive ego gift list has the implemented ego gift list. It takes the start ego gift, observed ego gift, and floor ego gift lists as arguments and highlight the gifts in the lists. Clicking unhighlighted ego gifts add their id into a list.. Hovering on a gift shows its available enhancement level, which is clickable. Clicking them selects the gift, if not selected, and shifts the enhencement level. Clicking the activated level unselects the gift. Hovering on a gift also shows a pane with enhencement level-wise ego gift deescription.

The floor section shows a floor-wise theme pack viewer on the left and a floor-wise gift list on the right. The number of floors are determined by the category (5F, 10F, 15F). The theme pack viewer consists of floor name (1F, 2F, ...) , theme pack image, and theme pack name, in vertical order. Clicking the theme pack image/name opens a theme pack selector pane. On the pane, clicking a theme pack sets the floor's theme pack. The gift list is the implemented ego gift list which takes a list of ids theme-pack specific ego gifts and common gifts as an argument. If a pack is not selected, show a i18n placeholder text on the center. 

Each section except category and name section has comment editor on the bottom of them. This reddit-like editor supports markdown, image upload, and draft saving. 

Clicking a save button saves the chosen gift/theme pack, name, category, deck (identity/ego), start buff, and the corresponding comments in the local IndexDB for a guest or to the server for a logined user.

Clicking a upload button sends the data to the server so that the planner is published in public.

For every change of id/ego/gift/theme pack and comment edit, save a draft on the local.
