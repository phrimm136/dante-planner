## Settled decisions (pinned into Briefs) — provisional, refined as Explore lands
- Storage keys (mechanics §1): `bl:<tokenHash>` string `SET … PEX <remaining-expiry>`; `uinv:<userId>` string epoch-ms `SET … PEX <refresh-token max life>`. Reuse the existing SHA-256 `hashToken`.
- Value payload (§9.1 PLAN-TIME): `bl:` value must encode the `immediate` flag AND `blacklistedAt` epoch-ms so the read-side 5s rotation-grace compare stays Java-side (rotation-blacklisted token allowed within 5s, rejected after; logout=immediate rejected instantly). Agent picks the exact encoding.
- Redis access (CONFIRMED via Explore): inject Spring Boot auto-configured `StringRedisTemplate` (no `@Qualifier` — resolves against the `@Primary` `authRedisConnectionFactory`), mirroring `shared/readpath/ContentTombstoneStore`. Write TTL via `opsForValue().set(key, value, Duration)` (Lettuce emits `SET … PX`); presence via `hasKey`/`opsForValue().get`. Constructor injection (CLAUDE.md — no field injection).
- Fail-open ladder (CONFIRMED pattern): catch exactly `org.springframework.dao.DataAccessException` (single catch, as ContentTombstoneStore) → on failure increment `blacklist_check_skipped_total` and return false (read proceeds). Counter idiom (from `PrimaryReCheck`): `static final String … = "blacklist_check_skipped_total"; Counter.builder(NAME).register(meterRegistry).increment();` — inline, no tags, injected `MeterRegistry`.
- Test-profile fact (CONFIRMED): `test`/`it` profiles set `management.health.redis.enabled=false`; plain `@SpringBootTest(MOCK)` has NO live Redis (Lettuce beans lazy, Redis collaborators mocked). So any suite exercising the REAL Redis-backed service needs a live test Redis wired via `@DynamicPropertySource`.
- Delete BOTH `@Scheduled` cleanup methods (`cleanupExpired`, `cleanupUserInvalidations`) and the `MAX_BLACKLIST_SIZE` eviction — TTL (PEX) replaces them.
- INV6 harness (PLAN-TIME, SPIKED): DegradationIT's auth Redis configured `--appendonly yes --appendfsync always --enable-debug-command local`. AOF-replay proven via admin `DEBUG LOADAOF` (reload dataset from AOF) — SPIKED against redis:7-alpine: default `DEBUG` is blocked ("ERR DEBUG command not allowed"); with `--enable-debug-command local` it returns OK and the `bl:` key (PX TTL) survives the reload. Deterministic (`always` fsync) — no wall-clock (INV4). Redis cut for fail-open via a Toxiproxy timeout toxic on a proxied auth-Redis path; cut MUST be restored in `@AfterEach` (toxic removal, not container stop) so the JVM-shared static auth Redis is not corrupted for sibling ITs — OR DegradationIT owns a dedicated auth-Redis container. tdd-red realizes the mechanism; asserted through control-API/condition.
- SCOPE BOUNDARY (advisor-pinned): Phase 5 is INV6 fail-open ONLY. Do NOT implement `AUTH_TEMPORARILY_UNAVAILABLE`/`WRITE_TEMPORARILY_UNAVAILABLE`, GlobalExceptionHandler, or JwtAuthenticationFilter Lettuce mapping — that is Phase 9 (status.json "Typed degradation + health semantics + Lettuce mapping"). Phase-5 fail-open is entirely internal to the blacklist service: catch Lettuce connectivity exception → increment counter → return false. Nothing propagates to handler/filter.

  - `TokenBlacklistServiceTest` (`service/token/`, plain JUnit, `new TokenBlacklistService()`; asserts `size()` many, `clear()` L165, `cleanupExpired()` L183/186, grace via `isBlacklisted`).   - Redis-SAFE (no change; run to confirm green): JwtAuthenticationFilterTest, JwtAuthenticationFilterLineageTest, AuthenticationFacadeTest, AuthenticationFacadeLineageTest, UserControllerTest (pure Mockito, no blacklist ref), UserAccountLifecycleServiceTest, AdminServiceTest — all `@Mock` the service; public method signatures are PRESERVED so mocks/verifies are unaffected.
