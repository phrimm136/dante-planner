# Research Document: Identity Browser UI Mock

## Overview of Codebase

- React 19 + Vite + Yarn with TanStack Router (v1.135.2) for file-based routing
- Tailwind CSS v4 with inline theme config, no separate config file required
- Shadcn UI components using Radix primitives for accessibility
- TanStack Query for data fetching with 1min staleTime, 5min cache, retry once
- i18n via react-i18next supporting EN/JP/KR/CN with single namespace pattern
- OAuth 2.0 authentication with Google, tokens stored in localStorage
- PascalCase naming for all files/components following {Entity}{Purpose} pattern
- Default exports preferred for page components, named for reusable components
- CVA (class-variance-authority) for component variant styling
- CSS variables for theming with light/dark mode via .dark class
- GlobalLayout wraps all pages with Header/main/Footer structure
- No existing filter/search/list components yet - clean slate
- Test setup with Vitest, renderWithProviders helper available
- API calls via ApiClient class with auth header injection
- Routes added programmatically via createRoute() and rootRoute.addChildren()
- Current routes: /, /about, /info, /planner, /community, /auth/callback/google

## Codebase Structure

- Routes in `frontend/src/routes/{Page}Page.tsx` with default exports
- Reusable components in `frontend/src/components/` (flat) and `components/ui/` (Shadcn)
- Feature components should go in `components/{feature}/` subdirectories
- Custom hooks in `frontend/src/hooks/use{Purpose}.ts`
- Types in `frontend/src/types/{entity}.ts`
- Services in `frontend/src/services/{entity}.ts`
- Router config at `frontend/src/lib/router.tsx`
- i18n config at `frontend/src/lib/i18n.ts`, translations at `/static/i18n/{LANG}/common.json`
- Global styles at `frontend/src/styles/globals.css` with @theme inline directive
- Context providers in `frontend/src/contexts/` (ThemeContext, AuthContext)
- Test utilities in `frontend/src/test-utils/`
- No nested routing currently, all routes are top-level under rootRoute
- Container pattern: `container mx-auto p-8` for page layouts
- Color tokens use oklch() color space with CSS custom properties

## Gotchas and Pitfalls

- routeTree.gen.ts is AUTO-GENERATED by Vite plugin - never edit manually
- Tailwind v4 uses @import not @tailwind directives, handled by Vite plugin
- Must add translations to ALL 4 languages (EN/JP/KR/CN) for complete UX
- Language codes are 2-letter UPPERCASE (EN not en)
- Query keys must include filter params for proper cache invalidation
- No Input component exists yet in ui/ - will need to create it
- All colors use CSS variables (--primary, --background) not hardcoded values
- Dark mode is class-based (.dark) not media query based
- Auth 401 errors auto-redirect to home via ApiClient
- Link component from TanStack Router needs mocking in tests
- Route paths must have leading slash: '/identity' not 'identity'
- Filter/search state changes require new queryKey to refetch data
- Tests use renderWithProviders not plain render() to include contexts
- Header navigation links are hardcoded - must manually add /identity link
- No mutations configured - only queries exist currently
- OAuth redirect URL hardcoded in Header component
