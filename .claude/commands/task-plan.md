# Implementation Plan
Based on the research.md findings, write a CONCISE plan (MAX 50 lines) to /$1/plan.md

**CRITICAL RULES**:
- NO code snippets or examples
- NO implementation details
- Use BULLET POINTS and numbered lists
- Be high-level and actionable
- **IDENTIFY and LIST any ambiguities or unclear requirements**

## Clarifications Needed (0-5 bullets)
**REQUIRED**: List any ambiguous requirements or implementation decisions that need user input:
- Component behavior and interaction patterns (e.g., selection logic, combination rules, default states)
- Default states or empty selection behavior
- Edge cases or error handling approaches
- UI/UX decisions not specified in requirements
- Performance or optimization trade-offs

**If no clarifications needed, explicitly state: "No clarifications needed - requirements are clear"**

## Task Overview (3-5 lines)
High-level summary of what will be implemented and why.

## Steps to Implementation (6-10 steps)
Numbered list of implementation steps:
1. **Step name**: Brief description (1-2 sentences max)
2. **Step name**: What needs to be done
3. Continue...

Each step should describe WHAT to do, not HOW (no code).

## Success Criteria (5-8 bullets)
Bullet list of what "done" means for this task.

## Assumptions Made (3-5 bullets)
**REQUIRED**: Explicitly list assumptions made during planning:
- Technical approach decisions
- Behavior choices where requirements were ambiguous
- Patterns or conventions being followed
- Any decisions that could reasonably be done differently
