# Execute the implementation plan for a given task

Read `$1/plan.md`, execute it, and document in `$1/code.md` (MAX 50 lines, NO code).

## Steps
1. **Read Context**: Read `$1/plan.md` (especially Scope section)
2. **Track Progress**: Create todos from plan steps. Mark in_progress → completed as you work.
3. **Execute**: Follow plan steps sequentially. Respect Scope constraints.
4. **Test**: Run Testing Guidelines from instructions.md
5. **Document**: Write to `$1/code.md` (MAX 50 lines, NO code snippets):
   - What Was Done (5-7 bullet points)
   - Files Changed (list paths only, no content)
   - What Was Skipped (2-3 bullets with reasons)
   - Testing Results (pass/fail summary only)
   - Issues & Resolutions (3-5 key items)

## Rules
- ONE todo in_progress at a time
- Stay within Scope boundaries
- Don't skip steps without good reason
- Document continuously, not at the end
- **Output Format**: Bullet points only, no code blocks
- **Length**: Each section max 10 lines
- **Focus**: What changed, not how it changed

ARGUMENTS: $1 (task directory path)
